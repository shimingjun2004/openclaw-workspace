## 系统定位

生产级中文发现系统 Stable v1.1 (2026-04-19)
定位：强发现层，弱正文层
目标：快、稳、智能、不折腾

## 架构原则（不可打破）

1. 不盲目扩源
2. 不引入重型浏览器链路（微信公众号/知乎正文是平台壁垒，非工程问题）
3. 发现层只做发现，正文按需单独调用
4. 分层路由，永不7路全开
5. 任何渠道故障不得拖垮整体

## 工具矩阵

cn_search.py        主入口，发现层       python3 cn_search.py <词>
chinese_hot.py       热点追踪           python3 chinese_hot.py weibo 10
zhihu_fetch.py       知乎按需专项       python3 zhihu_fetch.py <词>
wechat_fetch.py      微信按需专项       python3 wechat_fetch.py --sogou <词>
article_fetch.py     Playwright正文     python3 article_fetch.py <url>  # 公开网页
GitHub CLI           技术仓库精准搜索   gh search repos <词>

## 4种模式

hot    --hot                  微博+B站，10条/源，~0.3秒
fast   自动推断(4字内)         Tier-1单轮，3条/源，~0.8秒
normal 自动推断              Tier-1+智能Tier-2，5条/源，~1-3秒
deep   --deep                全链路+GitHub，8条/源，~4-5秒

模式推断逻辑：
深度词(分析/对比/横评/教程/最新动态...) -> deep
热点词(热搜/热点/突发...)              -> hot
短查询(4字以内)                         -> fast
其他                                    -> normal

## Tier路由

Tier-1(必须): 必应(Bing) + 360搜索 + 微博热搜 + B站热门
Tier-2(按需):  Tavily + B站视频 + 搜狗微信列表
Tier-3(deep): GitHub CLI

Tier-2触发条件(数量+质量联合判断):
1. 数量不足(< limit*1.5)
2. 来源不足(< 3个有效渠道)
3. 来源过于集中(某渠道 > 65%)
4. 标题重复率高(相似对 > 40%)
5. 360噪音过多(> 30%弱相关)
6. 查询意图偏深度(learn/compare/explain/news)

## 360噪音过滤

黑名单: AIDA、艾达、营销模型、注意力模型、销售模型、消费者行为
覆盖率: 标题含查询词 < 30% 即视为噪音
