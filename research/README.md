# 研究产出系统 — 命名规范与目录说明

## 目录结构

```
research/
├── tools/           # 工具评估、开源项目、技术方案
├── ai-trends/       # AI技术趋势、行业动态
├── building-ai/    # 建筑行业AI专项
├── college-tracking/# 高考/高校政策跟踪
├── weekly-reports/  # 周报
└── internal-notes/ # 内部笔记、想法
```

## 命名规则

```
YYYY-MM-DD-主题-类型.md
```

**类型后缀：**
- `-工具评估.md`
- `-技术调研.md`
- `-趋势简报.md`
- `-周报.md`
- `-笔记.md`
- `-版本评估.md`

**示例：**
- `2026-04-19-中文搜索系统-版本评估.md`
- `2026-04-19-中文搜索系统-观察记录.md`
- `2026-04-25-AI周报.md`
- `2026-04-26-建筑AI安全监管-趋势简报.md`
- `2026-04-26-高考志愿-跟踪记录.md`

## 3套模板

- `tools/工具评估-模板.md`
- `ai-trends/技术调研-模板.md`
- `ai-trends/趋势简报-模板.md`

## 使用原则

1. 有价值的调研 → 套模板保存
2. 随手聊天记录 → 不保存
3. 每周六整理本周研究 → 产出一份周报
4. 同一主题多次调研 → 追加到同一文件，不重复建档
